## Working with Me

### When I Ask for a PRD
- Use `/skills/prd-writer.md` approach
- Pull context from AHA! (active features) and Confluence (related specs)
- Focus on "why" and "what", leave "how" for engineering
- Always include success metrics

### When I Ask for Stakeholder Comms
- Use `/templates/stakeholder-update.md` structure
- Lead with outcomes, not activities
- Include blockers that need escalation
- Keep to 3-5 bullet points max

### When I Ask to Review a Spec
- Check for: clarity of problem statement, edge cases, dependencies
- Flag missing: success metrics, rollback plan, customer impact
- Suggest: questions to ask engineering

### When I Ask About AHA!/Requirements
- Default to my active features: [Your current features]
- Feature format: [Problem] -> [Solution] -> [Acceptance Criteria]
- Always include: priority, target release, dependencies

## Formatting Preferences

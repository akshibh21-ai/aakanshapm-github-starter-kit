# Feature Kickoff Workflow

## Purpose

End-to-end process to kick off a new feature from idea to ready-for-engineering.

## When to Run

- Starting a new feature or initiative
- After prioritization decision to build something
- Beginning of a quarter for planned features

## Prerequisites

- Feature idea or customer request approved for exploration
- Stakeholder alignment on priority
- Design and engineering bandwidth confirmed

---

## Steps

### Step 1: Gather Context

**Goal:** Collect all relevant inputs

**Actions:**
1. Pull customer feedback from Ideas Portal (use `customer-feedback-synth.md` if needed)
2. Review related Confluence specs
3. Check existing AHA! features for overlap
4. Identify stakeholders to involve

**Output:** Context document or notes

---

### Step 2: Write PRD

**Skill:** `skills/prd-writer.md`

**Input:**
- Problem statement from customer feedback
- Business context from stakeholder conversations
- Any existing solution ideas

**Output:** Draft PRD

**Checklist:**
- [ ] Problem statement is customer-centric
- [ ] Success metrics are measurable
- [ ] Scope is clear (in/out defined)

---

### Step 3: Create AHA! Feature

**Skill:** `skills/aha-ticket-manager.md`

**Input:**
- PRD from Step 2
- Priority recommendation
- Target release

**Output:** AHA! feature ready for prioritization

**Checklist:**
- [ ] Feature linked to PRD
- [ ] Acceptance criteria are testable
- [ ] Priority is documented

---

### Step 4: Request Design

**Actions:**
1. Share PRD with design
2. Schedule kickoff sync
3. Align on timeline for designs

**Output:** Design request submitted

**Checklist:**
- [ ] Designer assigned
- [ ] Design timeline agreed
- [ ] Key screens/flows identified

---

### Step 5: Kickoff Meeting

**Skill:** `skills/meeting-summarizer.md` (after meeting)

**Attendees:** PM, Engineering Lead, Designer

**Agenda:**
1. Walk through PRD (10 min)
2. Initial technical feasibility (10 min)
3. Design approach discussion (10 min)
4. Agree on next steps (5 min)

**Output:** Meeting summary with action items

**Checklist:**
- [ ] Engineering feasibility confirmed
- [ ] Design timeline confirmed
- [ ] Spec owner assigned
- [ ] Next checkpoint scheduled

---

### Step 6: Stakeholder Update

**Skill:** `skills/stakeholder-comms.md`

**Input:**
- Feature kicked off
- Timeline and owners
- Any risks identified

**Output:** Stakeholder notification

**Checklist:**
- [ ] Key stakeholders informed
- [ ] Timeline communicated
- [ ] Dependencies called out

---

## Workflow Checklist

- [ ] **Context gathered** - Customer evidence, related specs
- [ ] **PRD written** - Problem, solution, metrics defined
- [ ] **AHA! feature created** - Prioritized and linked
- [ ] **Design requested** - Designer assigned
- [ ] **Kickoff complete** - Team aligned, spec owner assigned
- [ ] **Stakeholders updated** - Timeline communicated

---

## Time Estimate

| Step | Typical Duration |
|------|------------------|
| Gather Context | 1-2 hours |
| Write PRD | 2-3 hours |
| Create AHA! Feature | 30 min |
| Request Design | 15 min |
| Kickoff Meeting | 30-45 min |
| Stakeholder Update | 15 min |

**Total:** ~1 day of focused work
